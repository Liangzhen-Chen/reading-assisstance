# ReadLens — AI Reading Companion

> AI 赋能阅读，而非替代阅读。

---

## 一句话定义

用户上传一本书（PDF/EPUB），在内置阅读器中阅读，AI 自动在段落旁生成轻量批注——标记核心论点、串联前后章节、梳理逻辑链——不打断阅读，不替代阅读。

---

## 问题

1. **信息密度高的书难以消化** — 读完一章记不清前面讲了什么，核心论点和举例展开混在一起
2. **现有 AI 工具的解法都有问题：**
   - NotebookLM 等全书总结工具 → 替代阅读，用户失去掌控感，不知道总结是否准确
   - 微信读书 AI 对话 → 打断阅读连贯性，边读边打字提问破坏体验
   - Readwise Ghostreader → 需要用户手动选中文字触发，不是自动伴随阅读
3. **Kairos（Every Studio, 2025）验证了需求但用错了形态** — chat-based 交互导致首日 1,000 下载、第三天零留存

## 解法

**自动生成的 inline 批注，贴在原文旁边，像一个读过这本书的人留下的铅笔笔记。**

用户不需要主动提问，不需要离开阅读界面。批注在阅读过程中自然存在，扫一眼就能获得额外的认知锚点。

---

## 核心用户

- 大学生（教材、学术书籍、专业读物）
- 知识工作者（商业/产品/思维类书籍）
- 共同特征：有阅读习惯，读信息密度高的非虚构类书籍，手上有 PDF/EPUB 文件

## 用户场景

> 我是 LSE 经济学学生，正在读 About Face（交互设计经典教材，700+ 页）。我上传 PDF 到 ReadLens，打开阅读器开始读。读到第三章时，段落旁边的批注告诉我「这是本章核心论点：目标导向设计的三个层次」，另一条批注说「这里的观点在第 7 章会被重新讨论并扩展」。我不用停下来整理笔记，也不用担心错过重点——AI 已经帮我标记了。

---

## 批注类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 🎯 核心论点 | 标记本段/本节的核心观点 | "本节核心：沉没成本不应影响决策" |
| 🔗 跨章节关联 | 指出与其他章节的联系 | "→ 第 7 章会用博弈论框架重新分析这个问题" |
| ⛓️ 逻辑链梳理 | 显式化隐含的因果推理链 | "论证链：A → B → C，其中 B 是关键假设" |
| ⚡ 速读提示 | 标记可以快读的段落 | "本段为举例展开，可快速浏览" |
| 📖 术语定义 | 首次出现的关键术语 | "边际效用：每增加一单位消费带来的额外满足感" |
| 🗺️ 位置感知 | 当前段落在全书论证中的位置 | "全书共 4 部分，当前位于第 2 部分（理论框架）" |

**批注密度原则：** 不是每段都有。只在需要的地方出现——大约每 2-3 页出现 1-2 条批注。过多会变成噪音。

## 产品形态

### Web App（MVP）

- **阅读器页面：** 左侧原文，右侧（或行间）显示 AI 批注
- **上传页面：** 拖拽上传 PDF/EPUB
- **首页：** 书架（已上传的书），阅读进度

### 技术方案（推荐）

| 层 | 选型 | 理由 |
|----|------|------|
| 前端框架 | Next.js (React) | 全栈框架，前后端统一 |
| PDF 渲染 | pdf.js | Mozilla 开源，成熟稳定 |
| EPUB 渲染 | epub.js | 社区主流方案 |
| AI API | Google Gemini Flash（免费层） | 免费额度充足，支持长上下文，用户无需配置 |
| 部署 | Vercel | 免费额度，git push 自动部署 |
| 数据存储 | 浏览器 IndexedDB | 纯本地存储，MVP 阶段不需要数据库 |

### 关键交互设计

1. **批注不需要点击展开** — 直接以浅色文字/小卡片形式显示在段落旁边
2. **批注可以折叠** — 用户如果觉得干扰，可以一键隐藏所有批注
3. **批注按章节预生成** — 用户翻到新章节时，后台预先生成该章节的批注（不是实时逐段生成）
4. **批注有 personality** — 语气像一个聪明的学长在旁边随手写的笔记，不是冷冰冰的 AI 输出

## 竞品定位

| 产品 | 做法 | 问题 |
|------|------|------|
| NotebookLM | 全书级 AI 总结 | 替代阅读，失去掌控感 |
| 微信读书 AI | Chat 对话 | 打断阅读流 |
| Readwise Ghostreader | 手动选中→触发 AI | 需要主动操作，非自动 |
| MarginNote | 手动批注+思维导图 | 非 AI 生成，手动工作量大 |
| Kairos (已关闭) | Chat-based AI 阅读 | 首日 1K 下载→第三天零留存 |
| **ReadLens** | **自动 inline 批注** | **无需操作，不打断阅读** |

## MVP 范围（2 周）

### Phase 1 — 能读（Day 1-4）
- [ ] Next.js 项目搭建 + Vercel 部署
- [ ] PDF 上传 + pdf.js 渲染
- [ ] 基础阅读器 UI（翻页、目录、进度）

### Phase 2 — 能批注（Day 5-9）
- [ ] AI 批注生成 pipeline（上传时按章节切分 → 调用 AI → 生成批注 JSON）
- [ ] 批注 UI（段落旁边显示，可折叠）
- [ ] 用量限制（每本书免费生成前 3 章批注，防止 API 滥用）

### Phase 3 — 能用（Day 10-14）
- [ ] Landing page
- [ ] 书架页面（多本书管理）
- [ ] 批注密度/风格设置
- [ ] README + 部署文档

### 暂不做
- EPUB 支持（MVP 只做 PDF）
- 用户账号系统（本地存储）
- 批注导出
- 移动端适配
